Re-pull loan data from public sources

Loan data is collected by an offline Python pipeline. It does not run in the browser.

Run these commands locally to refresh, then commit and push to redeploy:

cd dashboards/loan-maturity/data-collection
python pull_sba.py
python merge_loans.py
git add ../loans.json ../loan-maturity-tx.html
git commit -m "Refresh loan maturity data"
git push

What this dashboard shows

For every commercial loan in our 32-county Texas footprint that we can confirm from a free public record, the dashboard surfaces the borrower, lender, loan amount, approval date, and computed maturity date. The lens is prospecting: which loans are rolling off in the next 6-24 months, and who is the incumbent lender?

The Maturity Opportunity Score (0-100) blends time-to-maturity proximity, loan size, lender exit risk, geographic fit, borrower concentration, and source confidence. Each component is adjustable via the score-weights drawer.

Sources investigated but blocked (and why)
• State UCC bulk filings — TX SOSDirect requires paid subscription for bulk; per-debtor browser search is free but rate-limited. CA/FL/NY similar. Documented as enrichment-only.
• Dallas / Harris / Tarrant / Bexar / Travis County Clerks — All use vendor portals (publicsearch.us or equivalent). Free single-document download, but no CSV/JSON export. Bulk acquisition requires paid Data Sales contracts ($/county/quarter).
• Texas Open Data Portal (data.texas.gov) — No statewide commercial-loan or deed dataset published as of pull date.
• SEC EDGAR CMBS remittance reports (10-D) — Loan-level data exists in HTML/PDF exhibits but format varies per trustee. Implemented as a per-deal extraction in iteration 2.
